互联网医院项目技术说明文档

一、项目概述​

本项目是一个**互联网医院（Internet Hospital）**全栈应用，基于 RuoYi v3.9.0 框架构建。系统提供在线问诊、处方管理、支付处理、医院信息管理等完整的医疗健康服务。

属性说明
项目名称pro-internet-hospital
版本3.9.0
类型全栈医疗健康应用
部署方式前端 + 后端分离部署

二、项目组成​

本项目由 2 个代码仓库 组成：

pro-internet-hospital/
├── internet-hospital/                     # 后台管理系统（含 Java 后端 + Vue3 管理端）
│   └── ruoyi-ui/                          # Vue3 管理后台前端
└── OnlineInternetHospital_ReactTS_Frontend/  # 微信小程序用户端（独立仓库）
    └── src/                               # Taro 跨平台小程序源码

2.1 后台管理系统（internet-hospital）​

包含管理后台的 Java 服务端和 Vue3 前端：

模块说明
ruoyi-adminWeb 服务入口，启动端口 8002
ruoyi-framework框架核心（安全、异常、WebSocket 等）
ruoyi-system系统管理（用户、角色、菜单、字典、参数）
ruoyi-common通用工具类
ruoyi-quartz定时任务模块
ruoyi-generator代码生成器
ruoyi-consultation问诊模块
ruoyi-prescription处方模块
ruoyi-payment支付模块（微信支付、支付宝）
ruoyi-medical-card就诊卡模块
ruoyi-hospital医院信息管理
ruoyi-mini-app小程序模块
ruoyi-file-upload文件上传（七牛云存储）
ruoyi-doctor医生模块

2.2 微信小程序用户端（OnlineInternetHospital_ReactTS_Frontend）​

基于 Taro 的跨平台用户端小程序，支持微信、支付宝、百度等多平台。

五、核心功能模块​

5.1 问诊模块（ruoyi-consultation）​

支持多种问诊形式：

  • 图文问诊：文字+图片异步沟通
  • 视频问诊：实时视频通话（WebSocket 支持）
  • 健康讲座：医生直播课程

5.2 处方模块（ruoyi-prescription）​

  • 医生开具处方
  • 处方管理与审核
  • PDF 生成与导出

5.3 支付模块（ruoyi-payment）​

  • 微信支付：集成微信支付 v3 API
  • 支付宝支付：集成支付宝 SDK
  • 支付回调处理

5.4 预约挂号（registration）​

  • 科室选择
  • 医生排班
  • 挂号费支付

5.5 就诊卡（ruoyi-medical-card）​

  • 就诊卡绑定
  • 病史管理

六、API 架构​

6.1 管理后台 API​

控制器路径：/web/controller/

  • 使用 @RequiresPermissions 进行 RBAC 权限控制
  • 响应格式：AjaxResult

6.2 移动端 API​

控制器路径：/app/controller/

  • JWT Token 认证
  • WebSocket 支持实时通信

6.3 Swagger 文档​

开发环境访问：http://localhost:8002/swagger-ui/index.html

七、数据库​

7.1 主要数据库脚本​

  • sql/ry_20250522.sql - 主数据库（患者、医生、订单等）
  • sql/quartz.sql - 定时任务相关表

7.2 连接配置​

配置位置：ruoyi-admin/src/main/resources/application-*.yml

八、构建与运行​

8.1 后端构建​

# Windows 构建
cd internet-hospital
bin\package.bat

# 或 Maven 命令
mvn clean package -Dmaven.test.skip=true

# 运行
cd ruoyi-admin/target
java -jar ruoyi-admin.jar

8.2 管理前台构建​

cd internet-hospital/ruoyi-ui

# 安装依赖
npm install

# 开发环境
npm run dev

# 生产构建
npm run build:prod

8.3 小程序开发​

cd OnlineInternetHospital_ReactTS_Frontend

# 安装依赖
pnpm install

# 微信小程序开发
pnpm run dev:weapp

# H5 开发
pnpm run dev:h5

# 支付宝小程序
pnpm run dev:alipay

# 生产构建
pnpm run build:weapp
pnpm run build:h5

后端​

  • Maven 3.6+

  • Redis

  • Node.js >= 16
  • npm

前端（小程序）​

  • pnpm >= 7

开发工具​

场景工具
后端开发IntelliJ IDEA
前端开发VS Code
小程序开发微信开发者工具 / 支付宝开发者工具

十二、项目特点总结​

  1. 微内核 + 插件化模块设计：基于 RuoYi 框架，按业务拆分为独立 Maven 模块
  2. 前后端分离：管理后台 Vue3 SPA + 小程序跨端框架
  3. 双前端体系：
    • 管理后台：Vue3 + Element Plus（PC端）
    • 用户端：Taro + React（移动端小程序）
  4. 多端问诊支持：图文、视频、WebSocket 实时通信
  5. 完整支付体系：微信支付 v3 + 支付宝双渠道
  6. 处方电子化：在线开方、PDF 生成、药房配送
  7. 定时任务：Quartz 驱动的挂号提醒、订单超时等业务
