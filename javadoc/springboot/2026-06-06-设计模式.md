# 设计模式 —— Spring Boot 实战详解

> 设计模式是面向对象设计中**反复出现的问题**所对应的**可复用解决方案**。  
> 它们不是死板的代码模板，而是一种**组织类与对象关系**的思路。  
> Spring 框架本身大量运用了设计模式；在 Spring Boot 业务开发中，正确选用模式可以显著提升**可扩展性、可测试性和可维护性**。

## 创建型模式

> 关注**对象的创建过程**，将对象的创建与使用分离。

---

### 1. 单例模式（Singleton）

#### 意图

保证一个类在 JVM 中**只有一个实例**，并提供全局访问点。

#### Spring Boot 中的体现

Spring 容器管理的 Bean **默认就是单例**（`@Scope("singleton")`），整个应用共享同一实例。

```java
@Service  // 默认 singleton，全应用只有一个 UserService 实例
public class UserService {
    // ...
}
```

#### 需要多实例时：显式指定作用域

```java
@Component
@Scope(ConfigurableBeanFactory.SCOPE_PROTOTYPE)  // 每次注入/获取都 new 新实例
public class PrototypeBean { }

@Component
@Scope(WebApplicationContext.SCOPE_REQUEST)      // 每个 HTTP 请求一个实例
public class RequestScopedBean { }
```

#### 手写单例 vs Spring 单例

```java
// 手写双重检查锁 —— 业务代码中应避免，交给 Spring 管理
public class BadSingleton {
    private static volatile BadSingleton instance;
    public static BadSingleton getInstance() {
        if (instance == null) {
            synchronized (BadSingleton.class) {
                if (instance == null) {
                    instance = new BadSingleton();
                }
            }
        }
        return instance;
    }
}
```

**Spring Boot 实践建议：**

- 无状态 Service / Repository → 默认单例
- 有状态、线程不安全对象 → `prototype` 或 `request` 作用域
- 配置类 `@Configuration` 中 `@Bean` 默认也是单例

#### 应用场景

| 场景 | 示例 |
|------|------|
| 全局配置 | `AppProperties` 配置 Bean |
| 连接池 | `DataSource`、`RedisConnectionFactory` |
| 工具型无状态服务 | `IdGenerator`、`JwtTokenService` |

### 2. 工厂方法 / 简单工厂（Factory）

#### 意图

将对象的创建逻辑封装到工厂中，调用方只关心**接口**，不关心 `new` 哪个具体类。

#### Spring Boot 实战：按类型获取 Bean

Spring 容器本身就是**超级工厂**：

```java
@Service
@RequiredArgsConstructor
public class PaymentRouter {

    private final ApplicationContext context;

    public PaymentStrategy getStrategy(String payType) {
        // 从容器中按名称/类型获取对应实现
        return context.getBean(payType + "PaymentStrategy", PaymentStrategy.class);
    }
}
```

#### 简单工厂：集中创建逻辑

```java
@Component
public class NotificationFactory {

    public NotificationSender create(String channel) {
        return switch (channel) {
            case "email"    -> new EmailSender();
            case "sms"      -> new SmsSender();
            case "wechat"   -> new WechatSender();
            default -> throw new IllegalArgumentException("未知渠道: " + channel);
        };
    }
}
```

#### 工厂方法：每种产品一个工厂（更符合开闭原则）

```java
public interface NotificationFactory {
    boolean supports(String channel);
    NotificationSender create();
}

@Component
public class EmailNotificationFactory implements NotificationFactory {
    @Override
    public boolean supports(String channel) { return "email".equals(channel); }
    @Override
    public NotificationSender create() { return new EmailSender(); }
}

@Service
@RequiredArgsConstructor
public class NotificationService {
    private final List<NotificationFactory> factories;

    public void send(String channel, String message) {
        NotificationSender sender = factories.stream()
                .filter(f -> f.supports(channel))
                .findFirst()
                .orElseThrow()
                .create();
        sender.send(message);
    }
}
```

#### Spring Boot 自动配置就是工厂

```java
// Spring Boot 根据 classpath 自动"工厂生产" DataSource
@Configuration
@ConditionalOnClass(DataSource.class)
@ConditionalOnMissingBean
public class DataSourceAutoConfiguration {
    @Bean
    public DataSource dataSource(DataSourceProperties properties) {
        return DataSourceBuilder.create()
                .url(properties.getUrl())
                .username(properties.getUsername())
                .password(properties.getPassword())
                .build();
    }
}
```

### 3. 抽象工厂（Abstract Factory）

#### 意图

提供一个接口，用于创建**一族相关或相互依赖的对象**，而不指定它们的具体类。

#### Spring Boot 实战：多环境基础设施套件

```java
// 抽象工厂：定义一族产品
public interface InfrastructureFactory {
    CacheService createCache();
    MessageQueue createMessageQueue();
    FileStorage createFileStorage();
}

// 生产环境套件：Redis + RabbitMQ + OSS
@Component
@Profile("prod")
public class ProdInfrastructureFactory implements InfrastructureFactory {
    @Override public CacheService createCache() { return new RedisCacheService(); }
    @Override public MessageQueue createMessageQueue() { return new RabbitMqQueue(); }
    @Override public FileStorage createFileStorage() { return new OssFileStorage(); }
}

// 开发环境套件：本地缓存 + 内存队列 + 本地磁盘
@Component
@Profile("dev")
public class DevInfrastructureFactory implements InfrastructureFactory {
    @Override public CacheService createCache() { return new CaffeineCacheService(); }
    @Override public MessageQueue createMessageQueue() { return new InMemoryQueue(); }
    @Override public FileStorage createFileStorage() { return new LocalFileStorage(); }
}
```

业务层只依赖抽象工厂：

```java
@Service
@RequiredArgsConstructor
public class OrderService {
    private final InfrastructureFactory infrastructure;

    public void processOrder(Order order) {
        infrastructure.createCache().put("order:" + order.getId(), order);
        infrastructure.createMessageQueue().publish("order.created", order);
    }
}
```

#### 与工厂方法的区别

| | 工厂方法 | 抽象工厂 |
|--|---------|---------|
| 创建对象 | 一种产品 | 一族相关产品 |
| 典型场景 | 一种策略的多个实现 | 整套基础设施按环境切换 |

### 4. 建造者模式（Builder）

#### 意图

分步骤构建复杂对象，将**构建过程**与**表示**分离。

#### Spring Boot 常见用法

**1. Lombok `@Builder`（实体/DTO）**

```java
@Data
@Builder
public class SysConfigVO {
    private Long configId;
    private String configKey;
    private String configValue;
    private String remark;
}

// 使用
SysConfigVO vo = SysConfigVO.builder()
        .configId(1L)
        .configKey("sys.app.name")
        .configValue("super-system")
        .build();
```

**2. Spring Security 链式配置**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
                .csrf(AbstractHttpConfigurer::disable)
                .authorizeHttpRequests(auth -> auth
                        .requestMatchers("/public/**").permitAll()
                        .anyRequest().authenticated()
                )
                .sessionManagement(s -> s.sessionCreationPolicy(STATELESS))
                .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class)
                .build();  // 链式 build
    }
}
```

**3. `RestClient` / `WebClient` 构建**

```java
@Bean
public RestClient restClient(RestClient.Builder builder) {
    return builder
            .baseUrl("https://api.example.com")
            .defaultHeader("Accept", "application/json")
            .requestInterceptor((request, body, execution) -> {
                request.getHeaders().add("X-Request-Id", UUID.randomUUID().toString());
                return execution.execute(request, body);
            })
            .build();
}
```

**4. 自定义复杂对象构建器**

```java
public class ExportTask {
    private String format;
    private List<String> columns;
    private String filter;
    private boolean async;

    public static ExportTaskBuilder builder() { return new ExportTaskBuilder(); }

    public static class ExportTaskBuilder {
        private final ExportTask task = new ExportTask();
        public ExportTaskBuilder format(String format) { task.format = format; return this; }
        public ExportTaskBuilder columns(List<String> cols) { task.columns = cols; return this; }
        public ExportTaskBuilder filter(String filter) { task.filter = filter; return this; }
        public ExportTaskBuilder async(boolean async) { task.async = async; return this; }
        public ExportTask build() {
            Objects.requireNonNull(task.format, "format 不能为空");
            return task;
        }
    }
}
```

#### 适用场景

参数超过 4～5 个的构造器、可选参数多、对象创建后不可变（`record` + Builder）。

### 5. 原型模式（Prototype）

#### 意图

通过**复制现有对象**来创建新对象，而不是 `new`。

#### Spring Boot 实战

**1. `prototype` 作用域 Bean**

```java
@Component
@Scope(ConfigurableBeanFactory.SCOPE_PROTOTYPE)
public class ReportContext {
    private String reportType;
    private Map<String, Object> params = new HashMap<>();
    // 每个报表任务独立的上下文
}
```

**2. 深拷贝业务对象**

```java
@Service
public class OrderCloneService {

  /**
   * 基于已有订单快速创建「再来一单」
   */
    public Order cloneForReorder(Order source) {
        Order copy = new Order();
        copy.setUserId(source.getUserId());
        copy.setItems(source.getItems().stream()
                .map(item -> {
                    OrderItem newItem = new OrderItem();
                    newItem.setProductId(item.getProductId());
                    newItem.setQuantity(item.getQuantity());
                    return newItem;
                })
                .toList());
        return copy;
    }
}
```

**3. `ObjectMapper` 序列化克隆（简易原型）**

```java
@Service
@RequiredArgsConstructor
public class ConfigCloneService {
    private final ObjectMapper objectMapper;

    public <T> T deepClone(T source, Class<T> type) {
        return objectMapper.convertValue(source, type);
    }
}
```

### 6. 适配器模式（Adapter）

#### 意图

将一个类的接口转换成客户端期望的另一个接口，使原本不兼容的类可以协同工作。

#### Spring Boot 实战

**1. 第三方 SDK 适配为内部接口**

```java
// 内部统一接口
public interface SmsSender {
    void send(String phone, String content);
}

// 适配阿里云 SDK —— 将 Aliyun API 适配为 SmsSender
@Component
@ConditionalOnProperty(name = "sms.provider", havingValue = "aliyun")
public class AliyunSmsAdapter implements SmsSender {

    private final IAcsClient acsClient;  // 阿里云原生客户端

    @Override
    public void send(String phone, String content) {
        SendSmsRequest request = new SendSmsRequest();
        request.setPhoneNumbers(phone);
        request.setSignName("SuperApp");
        request.setTemplateCode("SMS_001");
        request.setTemplateParam("{\"code\":\"" + content + "\"}");
        acsClient.getAcsResponse(request);  // 适配翻译
    }
}
```

**2. Spring MVC `HandlerAdapter`**

Spring MVC 内部用适配器统一处理不同类型的 Controller：

- `@RequestMapping` 方法 → `RequestMappingHandlerAdapter`
- `HttpRequestHandler` → `HttpRequestHandlerAdapter`

开发者无感，框架帮我们做了适配。

**3. `WebMvcConfigurer` 适配旧接口**

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new AuthInterceptor()).addPathPatterns("/api/**");
    }
}
```

**4. 适配不同数据格式**

```java
@Component
public class LegacyUserAdapter {

    /** 将老系统 XML 用户数据适配为新系统 User 实体 */
    public User adaptFromLegacyXml(String xml) {
        LegacyUserXml legacy = parseXml(xml);
        User user = new User();
        user.setUsername(legacy.getLoginName());
        user.setPhone(legacy.getMobilePhone());
        return user;
    }
}
```

### 7. 装饰器模式（Decorator）

#### 意图

动态地给对象**添加额外职责**，比继承更灵活。

#### Spring Boot 实战

**1. Spring 缓存装饰**

```java
@Service
public class SysConfigServiceImpl extends ServiceImpl<SysConfigMapper, SysConfig> {

    // Spring Cache 在运行时"装饰"方法：先查缓存，未命中再执行原方法
    @Cacheable(value = "sysConfig", key = "#configKey")
    public SysConfig getByKey(String configKey) {
        return lambdaQuery().eq(SysConfig::getConfigKey, configKey).one();
    }

    @CacheEvict(value = "sysConfig", key = "#config.configKey")
    public void updateConfig(SysConfig config) {
        updateById(config);
    }
}
```

**2. 手动装饰：为 Service 增加日志/Metrics**

```java
public class MetricsSysConfigService implements SysConfigService {

    private final SysConfigService delegate;  // 被装饰的对象
    private final MeterRegistry meterRegistry;

  public MetricsSysConfigService(SysConfigService delegate, MeterRegistry registry) {
        this.delegate = delegate;
        this.meterRegistry = registry;
    }

    @Override
    public List<SysConfig> list() {
        return meterRegistry.timer("sysconfig.list").record(delegate::list);
    }
}

@Configuration
public class ServiceDecoratorConfig {
    @Bean
    @Primary
    public SysConfigService sysConfigService(SysConfigServiceImpl impl, MeterRegistry registry) {
        return new MetricsSysConfigService(impl, registry);
    }
}
```

**3. `HttpServletRequestWrapper` 装饰请求**

```java
public class RepeatableReadRequestWrapper extends HttpServletRequestWrapper {
    private final byte[] body;

    public RepeatableReadRequestWrapper(HttpServletRequest request) throws IOException {
        super(request);
        this.body = request.getInputStream().readAllBytes();
    }

    @Override
    public ServletInputStream getInputStream() {
        return new ServletInputStream() {
            private final ByteArrayInputStream bis = new ByteArrayInputStream(body);
            @Override public int read() { return bis.read(); }
            @Override public boolean isFinished() { return bis.available() == 0; }
            @Override public boolean isReady() { return true; }
            @Override public void setReadListener(ReadListener listener) { }
        };
    }
}
```

#### 装饰器 vs 代理

| | 装饰器 | 代理 |
|--|--------|------|
| 目的 | 增强功能 | 控制访问 |
| 关系 | 可多层叠加 | 通常一层 |
| 典型 | 缓存、日志、Metrics | 权限、事务、远程调用 |

### 8. 代理模式（Proxy）

#### 意图

为对象提供一个**代理**，以控制对这个对象的访问。

#### Spring Boot 核心：AOP 动态代理

Spring 的 `@Transactional`、`@Async`、`@Cacheable` 底层都是代理：

```java
@Service
public class OrderService {

    @Transactional(rollbackFor = Exception.class)  // 代理在方法前后管理事务
    public void createOrder(OrderDTO dto) {
        // 业务逻辑
    }

    @Async  // 代理将方法提交到线程池异步执行
    public void sendNotification(Long orderId) {
        // 发送通知
    }
}
```

#### 静态代理 vs 动态代理

```java
// JDK 动态代理示例（理解 Spring AOP 原理）
public class LoggingProxy implements InvocationHandler {
    private final Object target;

    public LoggingProxy(Object target) { this.target = target; }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        log.info("调用方法: {}", method.getName());
        Object result = method.invoke(target, args);
        log.info("方法返回: {}", result);
        return result;
    }
}
```

#### 自定义 AOP 切面（代理实战）

```java
@Aspect
@Component
@Slf4j
public class OperationLogAspect {

    @Around("@annotation(operationLog)")
    public Object around(ProceedingJoinPoint pjp, OperationLog operationLog) throws Throwable {
        long start = System.currentTimeMillis();
        try {
            Object result = pjp.proceed();
            log.info("[{}] 耗时 {}ms", operationLog.value(), System.currentTimeMillis() - start);
            return result;
        } catch (Exception e) {
            log.error("[{}] 异常: {}", operationLog.value(), e.getMessage());
            throw e;
        }
    }
}

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface OperationLog {
    String value();
}
```

#### Feign 远程代理

```java
@FeignClient(name = "user-service")
public interface UserClient {
    @GetMapping("/users/{id}")
    UserVO getUser(@PathVariable Long id);
}
// Feign 在运行时生成实现类，代理 HTTP 调用
```

### 9. 外观模式（Facade）

#### 意图

为子系统中的一组接口提供一个**统一的高层接口**，降低客户端与子系统的耦合。

#### Spring Boot 实战

**Service 层就是典型的 Facade：**

```java
@Service
@RequiredArgsConstructor
public class OrderFacade {

    private final OrderService orderService;
    private final InventoryService inventoryService;
    private final PaymentService paymentService;
    private final NotificationService notificationService;

    /**
     * 下单外观方法 —— 客户端（Controller）只需调这一个方法
     */
    @Transactional
    public OrderVO placeOrder(PlaceOrderDTO dto) {
        inventoryService.lockStock(dto.getItems());
        Order order = orderService.create(dto);
        paymentService.pay(order);
        notificationService.sendOrderConfirm(order);
        return OrderVO.from(order);
    }
}
```

Controller 保持简洁：

```java
@RestController
@RequiredArgsConstructor
public class OrderController {
    private final OrderFacade orderFacade;

    @PostMapping("/orders")
    public OrderVO placeOrder(@RequestBody PlaceOrderDTO dto) {
        return orderFacade.placeOrder(dto);  // 一个入口搞定
    }
}
```

**Spring Boot 内置 Facade：**

| 组件 | 外观对象 | 隐藏的子系统 |
|------|---------|-------------|
| `JdbcTemplate` | 简化 JDBC API | Connection、Statement、ResultSet |
| `SpringApplication.run()` | 启动入口 | 自动配置、环境准备、容器刷新 |
| `RestTemplate` / `RestClient` | HTTP 客户端 | 连接管理、序列化、异常处理 |

### 10. 组合模式（Composite）

#### 意图

将对象组合成**树形结构**，使客户端对单个对象和组合对象的使用保持一致。

#### Spring Boot 实战

**1. 菜单 / 权限树**

```java
@Data
public class MenuNode {
    private Long id;
    private String name;
    private String path;
    private List<MenuNode> children = new ArrayList<>();  // 组合子节点

    public void addChild(MenuNode child) {
        children.add(child);
    }

    /** 统一接口：获取所有叶子菜单的路径 */
    public List<String> collectLeafPaths() {
        if (children.isEmpty()) {
            return List.of(path);
        }
        return children.stream()
                .flatMap(child -> child.collectLeafPaths().stream())
                .toList();
    }
}

@Service
public class MenuService {
    public List<MenuNode> buildMenuTree() {
        List<Menu> all = menuMapper.selectList(null);
        return buildTree(all, 0L);  // 递归组树
    }

    private List<MenuNode> buildTree(List<Menu> menus, Long parentId) {
        return menus.stream()
                .filter(m -> m.getParentId().equals(parentId))
                .map(m -> {
                    MenuNode node = MenuNode.from(m);
                    node.setChildren(buildTree(menus, m.getId()));
                    return node;
                })
                .toList();
    }
}
```

**2. 组织架构 / 部门树**

```java
@Data
public class DeptTree {
    private Long deptId;
    private String deptName;
    private List<DeptTree> children;
}
```

**3. 文件目录结构**

与 `super-system` 中若依风格的 `sys_config` 树形分类类似，组合模式适合一切「父子层级」数据。

### 11. 桥接模式（Bridge）

#### 意图

将**抽象**与**实现**分离，使它们可以独立变化。

#### Spring Boot 实战

**消息发送：渠道（抽象）× 消息类型（实现）**

```java
// 抽象侧：消息类型
public abstract class AbstractMessage {
    protected MessageSender sender;  // 桥接到具体发送实现

    protected AbstractMessage(MessageSender sender) {
        this.sender = sender;
    }
    public abstract void send(String target, String content);
}

public class TextMessage extends AbstractMessage {
    public TextMessage(MessageSender sender) { super(sender); }
    @Override
    public void send(String target, String content) {
        sender.sendText(target, content);
    }
}

public class MarkdownMessage extends AbstractMessage {
    public MarkdownMessage(MessageSender sender) { super(sender); }
    @Override
    public void send(String target, String content) {
        sender.sendMarkdown(target, content);
    }
}

// 实现侧：发送渠道
public interface MessageSender {
    void sendText(String target, String content);
    void sendMarkdown(String target, String content);
}

@Component
public class DingTalkSender implements MessageSender { /* ... */ }

@Component
public class WechatWorkSender implements MessageSender { /* ... */ }
```

抽象（消息格式）和实现（发送渠道）可独立扩展，互不影响。

**JdbcTemplate 桥接数据库驱动：**

`JdbcTemplate`（抽象）桥接不同 `DataSource` 实现（HikariCP、Druid），业务 SQL 不感知底层连接池。

### 12. 享元模式（Flyweight）

#### 意图

运用共享技术有效地支持**大量细粒度对象**的复用，减少内存消耗。

#### Spring Boot 实战

**1. 配置数据缓存（运行时享元）**

```java
@Service
@RequiredArgsConstructor
public class SysConfigCacheService {
    private final SysConfigMapper configMapper;
    private final Map<String, SysConfig> configPool = new ConcurrentHashMap<>();

    @PostConstruct
    public void init() {
        configMapper.selectList(null)
                .forEach(cfg -> configPool.put(cfg.getConfigKey(), cfg));
    }

    /** 所有请求共享同一份 config 对象，而非每次查库 new */
    public String getValue(String key) {
        return configPool.get(key).getConfigValue();
    }
}
```

**2. `Integer` 缓存池（JDK 内置享元）**

```java
Integer a = 127;
Integer b = 127;
// a == b 为 true（-128~127 享元缓存）
```

**3. 连接池 / 线程池**

数据库连接、HTTP 连接、线程——都是享元思想：复用昂贵对象，而非每次创建。

**4. Spring 单例 Bean**

无状态 Bean 全应用共享一份，本质上也是享元。

### 13. 策略模式（Strategy）

#### 意图

定义一系列算法，将每个算法封装起来，并使它们**可以互相替换**。

#### Spring Boot 实战（最常用模式之一）

```java
public interface ExportStrategy {
    boolean supports(String format);
    void export(List<?> data, OutputStream out);
}

@Component
public class ExcelExportStrategy implements ExportStrategy {
    @Override public boolean supports(String format) { return "xlsx".equals(format); }
    @Override public void export(List<?> data, OutputStream out) { /* EasyExcel 导出 */ }
}

@Component
public class CsvExportStrategy implements ExportStrategy {
    @Override public boolean supports(String format) { return "csv".equals(format); }
    @Override public void export(List<?> data, OutputStream out) { /* CSV 导出 */ }
}

@Service
@RequiredArgsConstructor
public class ExportService {
    private final List<ExportStrategy> strategies;

    public void export(String format, List<?> data, OutputStream out) {
        strategies.stream()
                .filter(s -> s.supports(format))
                .findFirst()
                .orElseThrow(() -> new BusinessException("不支持的格式: " + format))
                .export(data, out);
    }
}
```

| 场景 | 策略接口 |

| 数据权限 | `DataScopeStrategy`（全部/本部门/仅本人） |
| 验证码 | `CaptchaStrategy`（图形/滑块/短信） |

**与 Spring `@ConditionalOnProperty` 结合：**

**2. 抽象任务处理器**

```java
public abstract class AbstractJobHandler {

    /** 模板方法：定义执行骨架，不允许子类覆盖 */
    public final void execute(JobContext context) {
        log.info("任务开始: {}", context.getJobName());
        try {
            validate(context);       // 钩子：子类可选实现
            doExecute(context);      // 抽象：子类必须实现
            onSuccess(context);      // 钩子：子类可选实现
        } catch (Exception e) {
            onFailure(context, e);   // 钩子
            throw e;
        } finally {
            cleanup(context);        // 钩子
        }
    }

    protected void validate(JobContext ctx) { }          // 默认空实现
    protected abstract void doExecute(JobContext ctx);   // 子类实现核心逻辑
    protected void onSuccess(JobContext ctx) { }
    protected void onFailure(JobContext ctx, Exception e) { log.error("任务失败", e); }
    protected void cleanup(JobContext ctx) { }
}

@Component
public class DataSyncJobHandler extends AbstractJobHandler {
    @Override
    protected void doExecute(JobContext ctx) {
        // 具体同步逻辑
    }
}
```

### 16. 责任链模式（Chain of Responsibility）

#### 意图

将请求沿着处理者链传递，直到有一个处理者处理它。

#### Spring Boot 实战

**1. Servlet Filter 链**

```
Request → CorsFilter → JwtFilter → AuthFilter → DispatcherServlet → Controller
```

```java
@Component
@Order(1)
public class TraceIdFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest req, HttpServletResponse res, FilterChain chain)
            throws ServletException, IOException {
        MDC.put("traceId", UUID.randomUUID().toString());
        try {
            chain.doFilter(req, res);  // 传递给链上下一个
        } finally {
            MDC.clear();
        }
    }
}

@Component
@Order(2)
public class RateLimitFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest req, HttpServletResponse res, FilterChain chain)
            throws ServletException, IOException {
        if (isRateLimited(req)) {
            res.setStatus(429);
            return;  // 中断链条
        }
        chain.doFilter(req, res);
    }
}
```

**2. Spring Security Filter Chain**

**3. 业务审批链**

```java
public interface ApprovalHandler {

    void handle(ApprovalContext context);

@Component

    @Override
    public void setNext(ApprovalHandler next) { this.next = next; }

    @Override
    public void handle(ApprovalContext ctx) {
        if (ctx.getAmount() <= 10000) {
            ctx.approve("部门经理");
            return;
        }
        if (next != null) next.handle(ctx);  // 传递给上级
    }
}

@Component
public class GeneralManagerHandler implements ApprovalHandler {
    @Override
    public void handle(ApprovalContext ctx) {
        if (ctx.getAmount() <= 100000) {
            ctx.approve("总经理");
        } else {
            ctx.reject("超出审批权限");
        }
    }
}
```

### 18. 状态模式（State）

#### 意图

允许对象在**内部状态改变时**改变其行为，对象看起来好像修改了它的类。

#### Spring Boot 实战：订单状态机

```java
public interface OrderState {
    void pay(OrderContext context);
    void ship(OrderContext context);
    void cancel(OrderContext context);
    String getStatus();
}

@Component
public class PendingPaymentState implements OrderState {
    @Override
    public void pay(OrderContext ctx) {
        ctx.setState(ctx.getPaidState());
        ctx.updateStatus("PAID");
    }
    @Override
    public void ship(OrderContext ctx) {
        throw new BusinessException("未支付，不能发货");
    }
    @Override
    public void cancel(OrderContext ctx) {
        ctx.setState(ctx.getCancelledState());
        ctx.updateStatus("CANCELLED");
    }
    @Override
    public String getStatus() { return "PENDING_PAYMENT"; }
}

@Service
@RequiredArgsConstructor
public class OrderStateMachine {
    private final PendingPaymentState pendingPaymentState;
    private final PaidState paidState;
    private final CancelledState cancelledState;

    public void pay(Long orderId) {
        OrderContext ctx = loadContext(orderId);
        ctx.getCurrentState().pay(ctx);
    }
}
```

public interface RuleExpression {
    boolean interpret(RuleContext context);
}

public class AndExpression implements RuleExpression {
    private final RuleExpression left;
    private final RuleExpression right;

    @Override
    public boolean interpret(RuleContext ctx) {
        return left.interpret(ctx) && right.interpret(ctx);
    }
}

public class AmountGtExpression implements RuleExpression {

    @Override

        return ctx.getAmount().compareTo(threshold) > 0;

## 模式选型决策树

```
需要创建对象？
├── 对象全局唯一 → 单例（Spring 默认 Bean）
├── 创建逻辑复杂、参数多 → 建造者（@Builder）
├── 需要按类型动态创建 → 工厂 / 策略 + List 注入
├── 需要一整套环境相关对象 → 抽象工厂（@Profile）
└── 需要复制已有对象 → 原型（prototype / 深拷贝）

需要组合类结构？
├── 接口不兼容 → 适配器
├── 动态增强功能 → 装饰器 / AOP 代理
├── 控制访问 → 代理（@Transactional）
├── 简化复杂子系统 → 外观（Service 层）
├── 树形结构 → 组合（菜单/部门树）
└── 抽象与实现独立变化 → 桥接

需要对象间协作？
├── 多种算法可切换 → 策略
├── 固定流程、可变步骤 → 模板方法
├── 事件通知、解耦 → 观察者（Spring Event）
├── 请求沿链处理 → 责任链（Filter）
├── 封装请求、异步/队列 → 命令（MQ）
├── 状态影响行为 → 状态模式
└── 对象间不直接引用 → 中介者（Event / MQ）
```

## 设计原则提醒

1. **不要为了用模式而用模式** —— 简单 `if-else` 能解决的问题不必上状态机。
2. **Spring 已内置大量模式** —— 优先用框架能力（Event、AOP、AutoConfiguration），再考虑手写。
3. **模式与 SOLID 协同** —— 策略/工厂支撑开闭原则；外观/适配器支撑单一职责；代理/装饰器支撑依赖倒置。
4. **从业务变化点出发** —— 选择「最可能扩展」的地方应用模式，而非处处套用。
